windows 本地调试或编译运行
	打开unity项目skgame
		选择菜单 “File”-> "Build Settings..." 弹出面板中检查是否为 PC，Mac&Linux平台，如果是，再执行以下操作
		如果不是，则选择 PC，Mac&Linux 平台点击切换
		
		project面板中输入“AppConst” 会自动VS打开Appconst.cs代码，把里面的[改成true] DebugMode = true;
		然后选择Unity左上菜单“LuaFramework” -> build windows resources -> 等待编译完成 
			-> (如果没有报错)运行unity ，看效果
			-> (如果报错或运行出错)，将错误红色信息发至技术群让相关人员解决
		
Android 版本发布
	打开unity项目skgame
		选择菜单 “File”-> "Build Settings..." 弹出面板中检查是否为 Android|ios 平台，如果是，再执行以下操作
		如果不是，则选择 Android|ios 平台 点击切换
		project面板中输入 “ AppConst ” 会自动VS打开Appconst.cs代码，把里面的[改成true] DebugMode = false;
		然后选择Unity左上菜单“LuaFramework” -> build android|iphone resources -> 等待编译完成
			-> (如果没有报错, 可能出现生成 Lua/xxxx/..文件说在使用中无法删除的报错，可以忽略) 
			-> (如果报错或运行出错)，将错误红色信息发至技术群让相关人员解决
		如果不想打包发布有热更新功能的程序，在 选择菜单 “File”-> "Build Settings..." ->"Player settings..."->Inspetor面板中，“Other Settings” -> 预编译处理（Script Define Symbols）中相应改成如下之一：
				安卓|iOS (把 ONLINE 去掉即为局域网测试更新资源， 但是，要在appconst.cs设置好能访问的路径位置)
				1、外网开发版本 ASYNC_MODE;ERROR_CHECK_MODE;ONLINE 或 ASYNC_MODE;ERROR_CHECK_MODE;ONLINE;ONLINE_KAIFA (带热更新)
				2、外网稳定版本 ASYNC_MODE;ERROR_CHECK_MODE;ONLINE;ONLINE_WENDING  (带热更新)
				3、外网开发版本 ASYNC_MODE;ERROR_CHECK_MODE;ONLINE;SyncLocal 或 ASYNC_MODE;ERROR_CHECK_MODE;ONLINE;ONLINE_KAIFA;SyncLocal (不带热更新)
				4、外网稳定版本 ASYNC_MODE;ERROR_CHECK_MODE;ONLINE;ONLINE_WENDING;SyncLocal  (不带热更新)

		前端热更新
		E:\server\PHPnow-1.5.6\htdocs\data\VersionTool.exe 不用填写任何信息
